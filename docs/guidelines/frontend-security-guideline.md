# Frontend Security Guideline

## 目的

フロントエンドを実装・運用する際に共通して守るべきセキュリティ原則をまとめた。

## シークレット管理

- API エンドポイント・トークン・シークレットは環境変数（`import.meta.env.*`）で管理し、コードやリポジトリにハードコードしない

## XSS 対策

- ユーザー入力を DOM に挿入する前にサニタイズし、XSS を防止する
- innerHTML への直接代入は原則使用しない。使用する場合は DOMPurify 等でサニタイズ済みコンテンツに限定する

## 外部リンク

- `target="_blank"` を持つ外部リンクには必ず `rel="noopener noreferrer"` を付与する

## 依存関係の管理

- 依存ライブラリの CVE を定期的に確認する
- Renovate / Dependabot などのツールで更新を自動化し、脆弱性パッチを迅速に適用できる体制を整える
