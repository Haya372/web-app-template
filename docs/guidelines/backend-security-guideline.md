# Backend Security Guideline

## 目的

バックエンドサービスを実装・運用する際に共通して守るべきセキュリティ原則をまとめた。

## 入力検証

- 入力値はドメイン層に渡す前に検証し、SQL / コマンドインジェクションや CSRF など基本的な攻撃ベクトルを防ぐ
- Value Object のコンストラクタでバリデーションを行い、不正な値がドメイン内部に入り込まないようにする

## データアクセス

- DB や外部 API へのアクセスはプリペアドステートメントや公式 SDK を使用し、クエリへの生文字列結合は禁止する
- 資格情報（DB パスワード、API キー、JWT シークレットなど）は環境変数または Secret Store で管理し、コードやリポジトリにハードコードしない

## 依存関係の管理

- 依存ライブラリの CVE を定期的に確認する
- Renovate / Dependabot などのツールで更新を自動化し、脆弱性パッチを迅速に適用できる体制を整える
