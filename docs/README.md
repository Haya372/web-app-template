# Docs

このディレクトリには、このリポジトリに関するドキュメントをまとめます。

## ディレクトリ構成と役割

### decisions/
設計上の重要な判断（ADR: Architecture Decision Records）を記録します。

- なぜその設計・方針を選んだのか
- どんな選択肢を検討したのか
- 採用によるトレードオフは何か

設計の理由や比較は **すべてここに集約**します。

---

### context/
このテンプレートリポジトリが想定している **前提・背景・用語** をまとめます。

- どのような Web アプリケーションを想定しているか
- 対象とする範囲 / しない範囲
- 用語や前提知識の整理

※ 設計判断そのものは `decisions/` に記載します。

---

### guidelines/
開発時のルールや指針をまとめます。

- ディレクトリ構成
- 命名規則
- コーディング規約
- PR / レビューの方針

※ ルールの背景や理由は `decisions/` を参照してください。

---

### operations/
運用に関するドキュメントをまとめます。

- CI / デプロイ
- マイグレーション
- 障害対応やロールバック

---

## このドキュメント群について

- **理由・比較・意思決定の記録**は `decisions/` に集約します
- それ以外のドキュメントは、参照しやすい形を優先します
- 内容はプロジェクトの成長に合わせて、必要になったタイミングで追加します

最初から完璧な状態を目指す必要はありません。
